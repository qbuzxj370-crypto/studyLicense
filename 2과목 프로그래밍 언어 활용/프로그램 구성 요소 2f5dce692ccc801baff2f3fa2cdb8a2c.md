# 프로그램 구성 요소

# 배열과 포인터

## 배열 (Array)

같은 자료형의 여러 데이터를 연속된 메모리 공간에 순서대로 저장하고, 인덱스(index) 를 통해 각 요소에 접근하는 자료구조

- 하나의 이름으로 여러 기억장소를 가리키기 때문에 배열에서 개별적인 요소들의 위치는 첨자(인덱스)를 이용하여 지정
- 배열은 변수명 뒤에 대괄호[]를 붙이고 그 안에 사용할 개수(배열의 길이)를 지정
- 배열의 위치는 0부터 시작
- 행 우선으로 데이터가 기억장소에 할당
- C언어에서는 배열 위치를 나타내는 첨자 없이 배열 이름을 사용하면 배열의 첫 번째 요소의 주소를 지정하는 것과 동일

### 1차원 배열

변수들을 일직선상의 개념으로 조합한 배열

**형식**

자료형 변수명 [개수];

### 2차원 배열

변수들을 평면, 즉 행과 열로 조합한 배열

**형식**

자료형 변수명[행개수][열개수]

### 배열의 초기화

배열 선언 시 초기값을 지정하는 과정

- 배열 선언 시 배열의 크기를 생략하는 경우 반드시 초기값을 지정해야 초기값을 지정한 개수 만큼의 배열이 선언
- 배열의 초기값을 ‘숫자,’로 지정하면 배열의 첫 번째 요소에는 지정한 숫자가 입력되고 나머지 요소는 0으로 초기화

### 문자열

**C언어**

- 큰따옴표(””)로 묶인 글자는 글자 수에 관계없이 문장열로 처리
- 문자열을 저장하는 자료형이 없기 때문에 배열 또는 포인터를 이용하여 처리
- **형식**
    
    char 배열명[크기] = “문자열”;
    
    char *변수명 = “문자열”;
    
    포인터 사용 시 저장되는 값은 문자열 시작 주소
    
- 배열에 문자열을 저장하면 문자열의 끝을 알리기 위한 널 문자(’\0’)가 자동으로 삽입
- 배열에 문자열을 저장할 때는 배열 선언 시 초기값으로 지정해야 하며 이미 선언된 배열에는 문자열 저장 불가
- 문자열 끝에 자동으로 널 문자가 삽입되므로 널 문지까지 고려하여 배열 크기 지정

**Java**

Java에서는 주소를 컨트롤하는 기능이 없기 때문에 포인터 변수를 이용 불가능

Java에서는 문자열을 처리 가능한 클래스를 제공

클래스를 제공하므로 그에 따른 속성과 메소드도 지원

## 포인터 (Pointer)

변수의 주소값(메모리 위치)을 다루기 위한 개념이며, C언어는 주소를 직접 다룰 수 있는 포인터 기능을 제공

- 포인터(pointer) : 메모리 위치를 가리키는 주소값
- 포인터 변수(pointer variable) : 그 주소값을 저장하는 변수
- 포인터 변수 사용 시에는 자료형을 쓰고 변수명 앞에 간접 연산자*를 기입
    - 자료형 *변수명
- 포인터 변수에 주소를 저장하기 위해 변수의 주소를 알아낼 때는 변수 앞에 번지 연산자 &을 기입
    - 포인터 변수명 = &변수명;
- 실행문에서 포인터가 가리키는 위치의 값을 역참조(간접 참조)하여 읽거나/쓴다.
- 포인터 변수를 제외하면 주소값 자체로 사용
- 포인터는 스택/힙/전역(정적) 영역 등 어디의 주소든 가리킬 수 있다.
- 특히 malloc/free 등의 동적 메모리 할당으로 얻은 힙 메모리 블록을 가리키는 데 자주 사용된다.

### 용도

- 연결된 자료구조(연결 리스트, 트리 등)를 구성하기 위해 사용
- 동적으로 할당된 메모리(힙 영역)의 데이터를 가리키고 관리하기 위해 사용
- 배열을 인수로 전달하기 위해 사용
- 문자열을 표현/처리할 때 사용
    - (C 문자열은 '\0'로 끝나는 char 배열이며 char*로 다루는 경우가 많음)
- 큰 데이터/구조체를 복사하지 않고 참조로 전달하여 비용을 줄이기 위해 사용
- 메모리 주소를 통해 특정 메모리 위치에 직접 접근해야 하는 저수준 작업에서 사용

### 포인터와 배열

배열은 포인터 변수에 저장한 후 포인터를 이용해 배열의 요소에 접근 가능

- 배열 위치를 나타내는 첨자를 생략하고 배열의 대표명만 지정하면 배열의 첫 번째 요소의 주소를 지칭
- 배열 요소에 대한 주소를 지정할 때는 일반 변수와 동일하게 & 연산자를 사용
- 배열의 요소가 포인터인 포인터형 배열 선언 가능

# 함수와 클래스

## 함수

C언어는 함수 지향 언어

- 처음 시작 시 입력하는 main()도 함수
- 함수의 종류로는 C언어에 내장되어 있는 내장 함수와 사용자가 만들어 사용하는 사용자 정의 함수로 구분

### 사용자 정의 함수 (User-defined Function)

프로그래머가 필요에 따라 직접 만드는 함수로, 특정 작업을 수행하는 코드 블록을 이름으로 추상화한 것

- 코드의 재사용성, 가독성, 유지보수성을 높임
- 큰 문제를 작은 단위로 나누는 분할 정복(모듈화)에 유리

**구성 요소**

- **반환형(Return Type)** : 함수가 돌려주는 값의 자료형 (void면 반환 없음)
- **함수명(Function Name)** : 함수 식별자
- **매개변수(Parameter)** : 입력값(0개 이상)
- **함수 몸체(Body)** : 실제 실행 코드
- **반환문(return)** : 반환형이 void가 아니면 보통 필요

**작동 원리 (매개변수 전달 방식)**

- C는 기본적으로 값 전달: 인자가 복사되어 매개변수로 들어감
    - 함수 안에서 매개변수를 바꿔도 호출자 변수는 그대로
- 호출자 값을 바꾸고 싶으면 주소(포인터)를 전달
    - `void inc(int *p) { (*p)++; }`

**주의점**

- **프로토타입 누락**: C에서는 선언 없이 호출하면 컴파일/링크 단계에서 문제가 나거나 경고가 뜰 수 있음(환경/표준에 따라)
- **반환형 불일치**: int 반환인데 return 누락 → 미정 동작/경고
- **스코프(scope)**: 함수 내부 지역변수는 함수가 끝나면 소멸
    - 지역변수의 주소를 반환/저장하면 위험(댕글링 포인터)
- **배열 인자 크기 정보가 사라짐**: int a[]로 받으면 길이 n을 별도로 받아야 안전
- **원본 수정 의도**가 있으면 문서/이름으로 드러내기(예: set, update, 포인터 사용)

## 클래스 (Class)

객체 생성을 위한 필드(속성)와 메소드(함수)를 정의하는 설계도

- Java는 아무리 작은 프로그램이라도 클래스 생성 필수

### 클래스 사용 순서

1. 클래스 이름을 정하고 객체 생성을 위한 필드와 메소드 정의
    - class 키워드 사용
2. 생성자 함수를 사용해 객체를 생성
    - new 키워드 사용
3. 생성된 객체들을 이용해 프로그램 작성
