# 웹 프로그래밍 기초

# HTML

## 개요

HTML(HyperText Markup Language)은 인터넷 표준 문서인 하이퍼텍스트 문서를 만들 때 사용하는 마크업 언어

- 웹 브라우저에 출력되는 화면은 모두 HTML로 작성된 문서이다.
- 태그로 구성되며 상위 태그 아래에 복수의 하위 태그가 있는 트리 구조
- 태그는 속성(Attribute)을 사용하여 기능을 구체화할 수 있으며 대•소문자를 구별하지 않는다.
- 속성에 값을 지정할 때는 일반적으로 큰따옴표(” “)로 묶는다.
- 값을 작은 따옴표(’ ‘)로 묶거나 그냥 값을 입력할 수도 있지만 값에 공백이 포함된 경우에는 반드시 따옴표 사용

**마크업 언어 (Markup Language)**

- 다른 문서의 처리를 위해 문서의 논리 구조나 체계를 정의하는 언어

**태그 (Tag)**

어떤 기능이나 모양 등을 정의하기 위한 꼬리표를 의미

## **기본 구조**

```html
<!doctype html>
<html lang="ko">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>문서 제목</title>

    <!-- CSS 연결 -->
    <link rel="stylesheet" href="styles.css" />

    <!-- JS (head에 둘 경우 defer 권장) -->
    <script src="app.js" defer></script>
  </head>

  <body>
    <header>헤더</header>

    <main>
      <h1>메인 제목</h1>
      <p>본문 내용</p>
    </main>

    <footer>푸터</footer>
  </body>
</html>

```

- `<!doctype html>`: 문서가 **HTML5**임을 브라우저에 알림으로써 표준 모드로 렌더링
- `<html>`: 문서의 최상위 루트 요소 lang 속성으로 문서의 기본 언어를 지정하여 접근성/검색엔진/번역에 도움
- `<head>`: 문서 설정/메타데이터 영역(화면에 출력 되지 않는다.)
    - `<meta charset="utf-8">`
        - 문자 인코딩 지정(한글 깨짐 방지)
    - `<meta name="viewport" ...>`
        - 모바일 화면에서 레이아웃/확대 비율 제어(반응형 필수)
    - `<title>`
        - 브라우저 탭 제목, 검색 결과 제목에 영향
- `<body>`: 실제 화면에 렌더링되는 콘텐츠 영역

### 작동 원리

- 브라우저는 HTML을 읽어 **DOM(Document Object Model) 트리**로 변환해 구조를 만든 뒤 렌더링
- 태그 중첩은 DOM의 부모-자식 관계가 된다.
- CSS는 DOM 요소에 스타일을 적용하고, JS는 DOM을 조작한다.

**CSS/JS 로딩 위치**

- CSS: 보통 `head`에서 로드
- JS: 보통 `</body>` 직전 또는 `head`에 둘 땐 `defer` 권장

## **기본 태그**

HTML 문서로 내용 출력 시 사용하는 기본적인 태그의 종류

### 미디어 태그

**<img>**

이미지 파일을 출력

관련 속성

- src
    - 이미지 파일이 저장된 경로와 파일 이름을 지정
    - 경로의 구분자로 ‘/’사용
- width, height
    - 이미지의 너비와 높이를 지정
    - 정수 입력 시 픽셀 단위로 지정
    - 백분율(%) 입력 시 입력한 비율대로 브라우저 창의 비율에 맞춰 너비와 높이 결정
- title
    - 이미지 위에 마우스 커서를 놓으면 출력될 텍스트 결정

**경로 지정 방식**

- 절대 경로 방식, 상대 경로 방식
- 웹에 저장된 이미지 출력 시 절대 경로 방식 사용
- PC에 저장된 이미지 출력 시 상대 경로 방식 사용
- 현재 작성 중인 HTML 문서와 같은 폴더에 저장된 이미지 출력 시 경로 없이 파일 이름만 입력

### 텍스트 및 문단 태그

**<h1>•••<h6>**

- 입력된 내용에 제목 스타일을 적용
- 제목 스타일은 h1 부터 h6까지 있으며 h뒤에 붙는 숫자가 클수록 글자 크기가 축소

**<br>**

- 줄바꿈 태그
- 키보드의 [Enter] 키와 같은 의미

**<p>**

- 입력된 내용을 하나의 문단(Paragraph)으로 지정
- 문단으로 지정되면 출력된 내용의 위와 아래에 여백이 삽입

### 텍스트 표현 태그

**<mark>**

- 태그 안의 내용을 형광팬으로 칠한것처럼 출력

**<small>**

- 태그 안의 내용을 일반 텍스트 크기보다 작은 크기로 출력

**<sub>**

- 태그 사이에 입력된 내용을 아래첨자로 출력

**<cite>**

- 태그 안의 내용을 기울림꼴 서체로 출력
- 책이나 음악, 영화, 그림 등 창작물의 제목 표시에 사용

### 문서 연결 태그

**<a>**

텍스트나 이미지를 클릭했을 때 연결할 URL을 설정

관련 속성

- href : 연결할 URL을 지정
- target
    - 웹페이지가 열리는 방식을 지정
    - 생략하거나 “_self” 입력 시 현재 창에 연결된 웹페이지 출력
    - “_brank” 입력 시 새로운 창에 연결된
- URL (Uniform Resource Locater)
    - 인터넷 상에 존재하는 각종 자원이 있는 위치를 나타내는 표준 주소 체계

### 목록 태그

HTML 문서에서 글 목록을 작성할 때 사용하는 태그의 종류

- 제목 있는 목록을 표시

- <dd> 태그에 제목의 하위 내용을 작성

## 문서 구성 요소

### HTML 프레임 (Frame)

한 브라우저 창(또는 영역)을 여러 구역으로 나눠 각 구역에 다른 문서/페이지를 표시하는 기능

**<frameset>**

- 화면을 분할한다.
- <frameset>•••</frameset> 태그 사이에는 분할한 프레임의 개수만큼 <frame> 태그를 사용
- 관련 속성
    - rows
        - 화면을 가로로 분할
        - 분할할 프레임들의 크기를 쉼표(,)로 구분하여 설정
    - cols
        - 화면을 세로로 분할
    - 크기 지정
        - 크기 단위로 정수(픽셀), 백분율, * 중 하나 사용 가능
        - * : 다른 영역의 크기를 제외한 나머지 크기로 프레임의 크기가 결정

**<frame>**

- 분할한 각각의 프레임에 출력할 HTML 문서를 지정
- <frameset>으로 분할한 영역에 <frame> 태그가 적용되는 순서
    - 화면을 가로로 분할한 경우 : 상→하
    - 화면을 세로로 분할한 경우 : 좌→우
- 분할한 프레임의 개수는 <frameset>의 rows나 cols 속성으로 확인 가능
- 관련 속성
    - src
        - 프레임에 출력할 문서/페이지 주소 설정

## 사용자 입력 처리

### HTML 폼 (Form)

사용자로부터 정보를 입력받고 입력받은 데이터를 서버로 전송하기 위해 사용하는 틀

**<form>**

- 사용자로부터 정보를 입력받는 틀
- 관련 속성
    - method : 데이터 전송 방식
        - get : 입력 받은 데이터를 URL애 첨부하여 전송
        - post : 입력 받은 데이터를 메세지 형태로 전송
    - action : 데이터를 전송할 URL을 설정

**<input>**

- 데이터 입력에 사용할 요소를 생성
- 관련 속성
    - type : 요소의 종류를 지정
        - text : 텍스트 상자를 의미하는 속성값
        - password : 암호 입력용 텍스트 상자를 의미하는 속성값으로 텍스트 상자에 입력되는 내용을 ‘*’로 출력
        - radio : 라디오 버튼을 지정하는 속성값
        - checkbox : 확인란을 지정하는 속성값
        - submit : 폼에 입력된 데이터를 전송하라는 의미의 속성값
    - name : 데이터를 참조할 때 사용할 이름을 설정
    - value : 기본값 설정
    - checked : type이 radio나 checkbox일 때 기본으로 선택되어야 할 항목을 지정
    - required : 반드시 입력되어야 하는 요소를 지정

# JavaScript

## **기본 문법**

JavaScript는 객체 지향의 스크립트 언어로 주로 웹페이지의 동작을 구현

- C언어 문법을 기반으로 제작된 언어이므로 제어문, 반복문 등의 함수 사용 방법이 C언어와 동일
- 변수는 자료형에 관계없이 var, let 예약어를 사용하여 선언
- html 문서에 적용하는 방법
    - <script> 태그 내에 JS문법에 맞춰 소스코드 작성
    - <script> 태그에 src 속성으로 코드가 작성된 .js 파일의 경로를 작성

## **입출력**

### **대화상자**

- 화면에서 데이터를 입력받거나 내용을 출력하는 용도로 사용하는 창
- 관련된 메소드는 window 객체에 정의되어 있으며 메소드 사용 시 객체명은 생략 가능
- 대화상자가 출력되면 대화상자 종료 시까지 웹페이지 조작 불가능
- window 객체
    - 웹 브라우저에 관련된 모든 요소를 정의하는 최상위 객체
    - 어디서든 접근 가능하여 ‘전역 객체’라고 불린다.
    - 원칙적으로 객체에 속한 메소드는 window.alert()와 같이 객체명과 메소드를 함께 작성해야 하나 window 객체에 한해서는 alert()와 같이 객체명 생략 가능

**확인 대화상자**

- confirm() 메소드 사용

**입력 대화상자**

- prompt() 메소드 사용

### 콘솔 (Console)

개발자 도구의 콘솔 탭의 기능을 사용할 수 있게 하는 전역 개체

- 웹 브라우저에서 개발자 도구 창의 콘솔 탭에서 결과 확인 가능
- 콘솔의 출력 메소드에는 log(), info(), warn(), error()가 있으며 사용 형식은 모두 동일
- 개발자 도구
    - 개발자들이 웹 브라우저에서 코드 분석, 디버깅, 성능 확인, 트래픽 분석, 보안 등의 기능을 구현할 수 있도록 제공하는 툴

## **배열 (Array)**

복수의 데이터를 하나의 이름으로 정의하여 사용하는 데이터의 집합

- 동일한 자료형만 저장 가능한 C나 Java의 배열과 달리 다양한 자료형 저장 가능
- 변수 선언 시 자료형에 관계없이 var, let 예약어 사용
- 변수명은 사용할 배열의 이름으로 사용자가 임의로 지정
- 초기값으로 지정한 개수대로 배열의 요소가 생성

### 배열 요소의 추가

배열명.unshift(값) 

- 배열의 첫 번째 요소를 추가하고 배열의 새로운 길이를 반환

배열명.push(값)

- 배열의 마지막에 하나 이상의 요소를 추가하고 배열의 새로운 길이를 반환

배열명[위치] = 값

### 배열 요소의 삭제

배열명.shift() 

- 배열의 첫 번째 요소를 제거하고 반환

배열명.pop()

- 배열의 마지막 요소를 제거하고 반환

배열명.splice(위치, 개수, 값1, 값2, •••) 

- 배열의 임의의 위치에서 요소를 제거하거나, 새 요소를 추가하거나, 교체
- 제거된 요소들이 담긴 새로운 배열을 반환

### 배열의 일부 요소 추출

배열명.slice(초기 위치:최종 위치)

- 배열의 초기 위치부터 최종 위치-1까지의 복사본을 새로운 배열로 반환

배열명.slice(최종 위치)

- 배열의 첫 번째 요소부터 최종 위치까지의 복사본을 새로운 배열로 반환

### 배열 요소를 문자열로 반환

배열명.join(구분자)

- 배열의 요소들을 구분자로 연결하여 문자열로 반환
